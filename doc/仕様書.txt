「やりたいことリスト100」Webアプリ 仕様書1. プロジェクト概要アプリ名: My Bucket List 100（やりたいことリスト100）目的: ユーザーが自分の夢や目標（やりたいこと）をデジタル上で手軽に管理し、モチベーションを維持するためのツールを提供する。コンセプト: 「デジタルなのにアナログな書き心地」。大学ノートに鉛筆で書き込むような体験をWebアプリで実現する。ターゲット: スマホやPCで手軽にリスト管理を行いたい個人ユーザー。2. システム要件プラットフォーム: Webブラウザ（モダンブラウザ対応：Chrome, Safari, Edge, Firefox）。デバイス対応: レスポンシブデザイン（スマートフォン、タブレット、PC）。PWA (Progressive Web App): 対応。ホーム画面に追加してネイティブアプリのように利用可能。データ保存: クライアントサイド（ブラウザ）のLocalStorageのみを使用。サーバーサイドの実装は不要。オフライン動作: PWAキャッシュまたはローカル実行により、オフラインでも閲覧・編集が可能であること。3. 機能要件詳細3.1 メイン画面（リスト表示・操作）ヘッダーメニューボタン: ハンバーガーアイコン（≡）。押下するとサイドメニュー（リスト一覧）を展開する。リストタイトル: 現在表示中のリスト名を表示。タップすることでリネーム（編集）が可能。件数バッジ: 「現在の登録数 / 100」を表示する。※内部的には999件まで登録可能だが、モチベーション維持のため「100」を目標値として表示。リストエリア表示形式: 大学ノート風の罫線上にアイテムを表示。自動採番: CSSカウンターを使用し、上から順に「#1.」「#2.」... と番号を自動付与。並び替え時も常に連番を維持する。アイテム表示: チェックボックス、テキスト、削除ボタン（ホバー/操作時）で構成。完了状態: チェックボックスをONにすると、テキストに取り消し線を引き、文字色を薄くする。編集機能: テキスト部分をタップすると編集ダイアログ（プロンプト）が表示され、内容を修正できる。削除機能: アイテム右側の「×」ボタンで削除確認ダイアログを表示し、承認後に削除する。入力エリア（フッター固定）テキストボックス: 新しい項目の入力用。追加ボタン: 入力内容をリスト末尾に追加する。制約: 最大999件まで登録可能。上限を超えた場合はアラートを表示。3.2 並び替え機能操作: リストアイテムを長押し（スマホ）またはドラッグ（PC）して順序を入れ替えることができる。ライブラリ: SortableJS を使用。挙動: ドラッグ終了時にデータ配列の順序を更新し、保存する。3.3 サイドメニュー（リスト管理）表示: 画面左側からスライドインするドロワーメニュー。背景はオーバーレイで暗転させる。リスト一覧: 作成済みのすべてのリスト名と、各リストのアイテム数を表示。切り替え: リスト名をタップすると、そのリストをメイン画面に表示する。新規作成: 「+ 新しいリストを作成」ボタンで、新しい空のリストを作成する。名前の入力が必要。リスト削除: リスト名の横にある「×」ボタンで削除確認を行い、リストごとデータを削除する。（※リストが残り1つの場合は削除不可）3.4 データ管理機能自動保存: アイテムの追加、編集、削除、並び替え、リスト切り替え等の操作が行われるたびに、即座にLocalStorageへ保存する。データ移行: 旧バージョン（配列のみのデータ構造）が存在する場合、初回読み込み時に自動的に新データ構造（複数リスト対応）へマイグレーションを行う。4. UI/UXデザイン仕様4.1 デザインテーマメタファー: アナログのノートブック。フォント: Yomogi (Google Fonts) を使用し、手書き感を演出。4.2 カラーパレット項目色コード用途背景色#fcfbf7ノートの紙の色（クリーム色）アプリ外背景#e8e8e8PC表示時のウィンドウ外背景罫線色#a4b0beノートの罫線文字色#4b4b4bメインテキスト（濃い鉛筆色）アクセント#ff6b6b取り消し線（赤ペン）、強調補助色#ccc / #888プレースホルダー、番号、削除ボタン4.3 インタラクションタッチ操作: スマホでの操作を優先し、ボタンやタップ領域を広めに確保（40px以上推奨）。長押し防止: ドラッグ操作を阻害しないよう、user-select: none や -webkit-touch-callout: none を適用。5. データモデル設計LocalStorageキー: my_bucket_list_data保存データ構造 (JSON){
